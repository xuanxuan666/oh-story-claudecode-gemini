# 叙事单元体系

从原型故事中逐级提炼出的可复用写作工具。核心原则：**戏剧性会磨损，情绪不会磨损**。

## 提炼层级

| 层级 | 名称 | 内容 | 说明 |
|------|------|------|------|
| 零级 | 具体故事 | 原文完整情节（含所有描述和素材） | 直接搬用 = 抄袭 |
| 一级 | 经典故事情节 | 去除描写，保留核心素材（人名/地名/事件） | 换汤不换药，仍有雷同风险 |
| 二级 | 故事构型 | 去除所有素材，只剩逻辑走向 | 戏剧单元 + 情绪模块的组合体 |
| 三级 | 戏剧单元 | 戏剧性的抽象——有戏剧性的大致走向 | 单元化逻辑结构，可自由组合 |
| 四级 | 情绪模块 | 情绪的抽象——读者想看什么 | 由期待感挖掘方向 + 爽感挖掘方向组成 |

**故事构型 + 素材 = 具体故事**

## 核心概念

### 情绪模块（四级）

读者情绪需求的抽象。每个情绪模块对应一种读者想体验的核心情绪。

- **情绪不会磨损**：英雄救美从史诗时代到今天，情绪本身是长期有效的底层工具
- 解决"目标读者要获得什么情绪"
- 总结够多后组建情绪模块系统，指导情绪设计
- 具体：英雄救美、装逼、以小搏大、少年热血、临危受命、好人好报、点石成金、慧眼识真、歪打正着等（详见各卡具体定义）

### 戏剧单元（三级）

戏剧性走向的抽象。笼罩整本书或大段剧情的基调。

- **戏剧性会磨损**：同类结构用多了不好看
- 解决"当前 artifact 能生成什么剧情"
- 总结够多后组建戏剧单元系统，自由组合
- 戏剧性磨损后，可通过更换/强化对应情绪模块重组新结构
- 具体：恩怨情仇、修罗场（作为中戏剧单元）等

### 叙事元素

可插入故事的特定元素。不能独立提供完整故事框架，必须应用在某个完整框架内。

- 提供故事内的特殊元素：工具人、迷之少女、失能（限制条件）、关系、秘密、闹矛盾、双界联系、紧追其后、隐蔽行动、未来窥视、禁忌、新环境扮演/伪装/试探等

### 结构技法

对故事框架的调整思路。不能独立提供完整故事框架，必须应用在某个完整框架内。

- 提供结构性调整方法：差异评价、戏剧三角、翻弄风云、解谜设计等

## 戏剧单元 vs 情绪模块

| 维度 | 戏剧单元 | 情绪模块 |
|------|---------|--------|
| 本质 | 构建戏剧性的逻辑结构 | 读者想看什么的情绪需求 |
| 磨损 | 戏剧性会磨损 | 情绪不会磨损 |
| 功能 | 解决"当前 artifact 能生成什么剧情" | 解决"目标读者要获得什么情绪" |
| 修复 | 磨损后更换/强化对应情绪模块 | 长期有效，无需修复 |

## 使用方式

**1. 重构使用**：保持原有素材不变，彻底修改故事构型（更换/增加/减少情绪模块或戏剧单元），使观感完全不同。适用于长篇（百万字级别必须掌握）。

**2. 微调使用**：直接使用经典故事情节，但深挖人物人设，根据人设微调情节。适用于短篇。

**规则**：当人设/核心设定与经典故事情节有冲突时，必须重构使用。

## 组合规则

- 情绪模块之间可自由组合嵌套，制造多层爽感
- 情绪模块 + 叙事元素 + 结构技法 = 一个完整的剧情单元
- 上升型模块可单独使用（英雄救美、装逼）
- 下行型模块后面必须接上升型模块（陷入困境→装逼）
- 组合效果 > 单独使用（以小搏大+英雄救美=双重爽感）
- 同一套路每次出现时复杂程度和收获都上升（螺旋上升）
- 技巧服务表达，没用的技巧就丢弃

## 常见误区

1. **故事雷同**：只做一级提炼（换名字不换结构），读者眼里和抄袭无区别
2. **人设崩坏**：为了套用经典情节强行让人物做不符合人设的事
3. **创新失控**：脱离情绪模块的"创新"容易变成毒点
4. **扼杀核心创意**：要读者爱看还是要自己爱写变成二选一
5. **无法兼容文学性**：经典情节模板没有文学性插足空间
